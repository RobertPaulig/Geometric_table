# Протокол коммуникации (обязательный)

Назначение: сделать отчёты и постановки задач **машинно-одинаковыми**, чтобы STOP/GO работал без “домыслов”.

## 1) Шапка сообщения (обязательная)

Первая строка всегда в одном из видов:

- `ИСПОЛНИТЕЛЬ → АРХИТЕКТОРУ`
- `АРХИТЕКТОР → ИСПОЛНИТЕЛЮ`
- (если нужно) `АРХИТЕКТОР → ЧЕЛОВЕКУ`

Вторая строка: **контекст/файлы**, например:

`docs\\99_index.md / CONTEXT.md / docs\\name3.md / tests\\test_docs_index_integrity.py`

## 2) Отчёт Исполнителя (строго 4 блока)

Каждый ответ Исполнителя содержит **ровно** 4 блока (в таком порядке):

1. `Gate-status (STOP/GO)`
2. `Что сделал`
3. `Как проверил`
4. `Ссылки/артефакты`

## 3) Gate-status: что считать “точным SHA”

- Требуемые контексты: `ci/test`, `ci/test-chem`, `ci/docker`.
- Для PR “tested SHA” может быть **merge-ref SHA** (если workflow постит статусы на него).
- В Gate-status для PR всегда указываем:
  - `head SHA`
  - `tested SHA (merge-ref)`
  - ссылки на Actions run(ы)
- После мержа “истина” = **merge SHA в `main`**, и на нём обязаны быть 3 зелёных контекста.

Каноническая формулировка гейтов: `CONTEXT.md` → `## Gates (STOP/GO)`.
